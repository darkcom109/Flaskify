Lesson 18: Blueprints (Modular Apps)

In this lesson, youâ€™ll learn how to organize your Flask project using Blueprints, which let you split routes into separate modules.

Step 1: What Are Blueprints?

A Blueprint is a way to group related routes and logic. Instead of dumping everything into app.py, you can keep your app modular and organized.

Step 2: Create a Blueprint


    {% raw %}
    # blog/routes.py
    from flask import Blueprint

    blog = Blueprint("blog", __name__)

    @blog.route("/blog")
    def blog_home():
        return "Welcome to the blog!"
    {% endraw %}

Step 3: Register the Blueprint


    {% raw %}
    # app.py
    from flask import Flask
    from blog.routes import blog

    app = Flask(__name__)
    app.register_blueprint(blog)
    {% endraw %}

Now visiting /blog will hit the blueprint route.

Step 4: Using URL Prefixes


    {% raw %}
    # app.py
    app.register_blueprint(blog, url_prefix="/posts")
    {% endraw %}

Now your routes are available under /posts/....

Step 5: Multiple Blueprints


    {% raw %}
    # auth/routes.py
    from flask import Blueprint

    auth = Blueprint("auth", __name__)

    @auth.route("/login")
    def login():
        return "Login Page"

    # app.py
    from auth.routes import auth
    app.register_blueprint(auth, url_prefix="/auth")
    {% endraw %}

You can now visit /auth/login for the auth blueprint.

Step 6: Why This Matters

Blueprints make large projects manageable by splitting features into separate files and folders. This is essential once you start building real-world apps.
