Lesson 14: Jinja Macros & Template Inheritance

In this lesson, you’ll learn how to reduce repetition in your templates using template inheritance and Jinja macros.

Step 1: What Is Template Inheritance?

Instead of writing the same HTML structure (head, navbar, footer) on every page, you create a base.html template and let other templates extend it. 


{% raw %}




    Flaskify


    
Navbar here


    {% block content %}{% endblock %}

    
Footer here



{% endraw %}

Step 2: Extending a Template


{% raw %}

{% extends "base.html" %}

{% block content %}
    
Welcome to Flaskify!

    
Learning Flask through Flask.

{% endblock %}
{% endraw %}

This keeps your HTML consistent and much easier to maintain.

Step 3: Using Jinja Macros

A macro is like a function inside your template. It lets you reuse small chunks of code. 


{% raw %}

{% macro render_input(name, type="text") %}
    
    
{% endmacro %}
{% endraw %}

Step 4: Importing and Using a Macro


{% raw %}

{% from "macros.html" import render_input %}



    {{ render_input("username") }}
    {{ render_input("password", type="password") }}
    


{% endraw %}

Now you don’t have to copy-paste form fields everywhere — just call the macro.

Step 5: Combining Techniques

Use template inheritance for big page layouts and macros for small reusable components. Together, they keep your project clean and DRY (Don’t Repeat Yourself).
