Lesson 13: Custom Error Pages & Handling

In this lesson, you’ll learn how to replace Flask’s default error messages with your own custom pages.

Step 1: Why Customize Error Pages?

Flask’s default errors are plain and ugly. Custom pages improve user experience and keep your app looking polished — even when something goes wrong.

Step 2: Add Error Handlers


        from flask import render_template

        @app.errorhandler(404)
        def page_not_found(e):
            return render_template("404.html"), 404

        @app.errorhandler(500)
        def internal_error(e):
            return render_template("500.html"), 500

Step 3: Create Templates


        
        <h1>404 - Page Not Found</h1>
        <p>Oops! Looks like this page doesn’t exist.</p>

        
        <h1>500 - Server Error</h1>
        <p>Something went wrong on our end. Please try again later.</p>

Step 4: Test It Out

- Visit a route that doesn’t exist → you’ll see your new 404 page. - Trigger an internal error (like dividing by zero in a view) → you’ll see the 500 page.

Step 5: Extra Polishing

Add your site’s navbar, styles, and branding to the error pages so they feel consistent with the rest of your app.
