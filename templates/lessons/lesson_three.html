Lesson 3: Templates & Jinja Basics

In this lesson, you’ll learn how Flask uses templates to render HTML and how Jinja2 lets you add dynamic content to your pages.

Step 1: Using a Template


    # app.py
    from flask import Flask, render_template

    app = Flask(__name__)

    @app.route("/")
    def home():
        return render_template("home.html")

Instead of returning a raw string, Flask uses render_template() to return an HTML file. By default, Flask looks for templates in a folder called templates/.

Step 2: Creating home.html


    <!DOCTYPE html>
    <html>
    <head>
        <title>My Flask App</title>
    </head>
    <body>
        <h1>Welcome to Flaskify!</h1>
        <p>This page is rendered from a template.</p>
    </body>
    </html>

Step 3: Passing Data to Templates


    # app.py
    @app.route("/user/<name>")
    def user(name):
        return render_template("user.html", username=name)
    

    <h1>Hello, {{ "{{ username }}" }}!</h1>
    <p>Welcome to Flaskify.</p>

Jinja placeholders {{ "{{ ... }}" }} let you insert Python values directly into HTML. Visiting /user/Alice will show “Hello, Alice!”

Step 4: Logic in Templates


    <h1>Your To-Do List</h1>
    <ul>
    {% raw %}{% for item in items %}{% endraw %}
        <li>{{ "{{ item }}" }}</li>
    {% raw %}{% endfor %}{% endraw %}
    </ul>
    

    # app.py
    @app.route("/todos")
    def todos():
        tasks = ["Learn Flask", "Build a project", "Celebrate"]
        return render_template("list.html", items=tasks)

Jinja uses tags like {% raw %}{% for ... %}{% endfor %}{% endraw %} and {% raw %}{% if ... %}{% endif %}{% endraw %} to add loops and conditionals. This lets you dynamically generate lists, tables, or other repeated content.
